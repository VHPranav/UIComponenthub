## Overview

**Number Count Up** is a performant component designed for dashboard metrics and data visualizations. Unlike basic interval-based counters, it uses a **physics-based spring** from Framer Motion. This means the speed of the counting feels natural, accelerating at the start and gracefully slowing down as it reaches the final value.

## Features

- **Spring Physics**: Uses `damping` and `stiffness` to create a more organic numerical transition.
- **Decimal Precision**: Automatically detects and respects the number of decimal places in your target value.
- **Trigger on Visibility**: Uses `useInView` to automatically start counting when the user scrolls to the section.
- **Custom Formatting**: Easily add separators (commas, dots) for professional financial or data displays.
- **Event Hooks**: `onStart` and `onEnd` callbacks for triggering secondary effects or sequences.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `to` | `number` | `100` | The target value to count up to. |
| `from` | `number` | `0` | The starting value for the animation. |
| `duration` | `number` | `2` | Estimated duration in seconds (affects spring stiffness). |
| `delay` | `number` | `0` | Delay in seconds before starting the count. |
| `separator` | `string` | `""` | The character used for grouping thousands (e.g. `","`). |
| `startWhen` | `boolean` | `true` | Condition that must be true for the animation to begin. |
| `onStart` | `() => void` | `undefined` | Callback fired when counting begins. |
| `onEnd` | `() => void` | `undefined` | Callback fired when counting is complete. |

## Technical Details

The component updates the `textContent` of a `span` ref directly using Framer Motion's `springValue.on('change')` listener. This bypasses React's state-update overhead for high-frequency numerical shifts, ensuring 60fps performance even with large numbers or multiple counters on a single page.
